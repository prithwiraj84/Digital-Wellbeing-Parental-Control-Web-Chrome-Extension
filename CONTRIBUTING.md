# Contributing

Thanks for your interest in contributing to this project — we appreciate it!

How to contribute
-----------------
1. Fork the repository and create a branch with a descriptive name, e.g. `feature/settings-ui`.
2. Make small, focused commits. Use clear messages describing what and why.
3. Push your branch and open a Pull Request to the `main` branch.
4. Describe your changes in the PR, link related issues, and add screenshots if your change affects UI.

Code style
----------
- Keep JavaScript simple and well-documented.
- Follow existing project conventions for naming and structure.
- Run any linters or formatters you use locally before submitting.

Testing
-------
- Manually test UI changes by loading the extension unpacked (see `docs/INSTALL.md`).
- Provide steps to reproduce bugs in issues and PR descriptions.

Reporting security issues
------------------------
If you find a security vulnerability, please open a private issue or contact the maintainers directly instead of posting it publicly.

Thank you — your contributions make this project better!
